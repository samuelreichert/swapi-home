## Overview

Three-page SPA that lets users search Star Wars characters by name. Search input is debounced at 750ms to minimize API calls. Results are limited to 5 per query. The app is deployed on Vercel with SPA rewrite rules so all routes are handled client-side.

## Architecture

**Data fetching:** TanStack Query v5 drives the search with `enabled` tied directly to the debounced input value — no manual `refetch()` calls or `useEffect` orchestration. The query re-runs automatically when `debouncedSearch` changes and is non-empty.

**Component model:** A `withPageLayout` higher-order component wraps each page with the shared header, navigation, and footer, keeping page components focused on their own content.

**Styling:** No CSS-in-JS runtime. Olyx UI provides accessible, unstyled Base UI primitives with native CSS custom properties. Brand color tokens are defined in `index.css` and consumed via `var(--color-*)` throughout. CSS layers (`@layer reset, base, tokens`) prevent specificity conflicts.

**Type safety:** Strict TypeScript throughout. `QueryFunctionContext` types the query key tuple in `searchCharacters.ts`. Generic constraint `<T extends object>` on the HOC satisfies the JSX generic disambiguation requirement in TS 5.

## Setup

```bash
git clone https://github.com/samuelreichert/swapi-home
cd swapi-home
npm install
```

## Functional Requirements

- Search results capped at 5 characters per query
- Debounce at 750ms minimizes API calls while typing
- Three pages: Home, Characters, About
- Shared header, navigation, and footer across all pages

## Deployment

Configured for Vercel. `vercel.json` rewrites all routes to `index.html` so React Router handles navigation client-side. Deploy by connecting the repository to a Vercel project — no additional configuration needed.
